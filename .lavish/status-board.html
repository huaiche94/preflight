Tier 1 — 已接線且通過測試（今天就在 shipped binary）

能力證據 / 為何算數參照
決策面 Decision surface policy engine 產生並記錄 CHECKPOINT_AND_RUN 等決策（WARN · REQUIRE_CONFIRMATION · SPLIT · PAUSE · PAUSE_AND_AUTO_RESUME · BLOCK），透過 hook response 回傳給 agent。 real
證據閘 Evidence gate work unit 沒有驗證過的證據（file / DB / checksum / Git snapshot）就不算完成；「agent 說做完了」永遠不算。閘門活在 context window 之外，不會遺忘。 real
Progress Tree canonical、durable 的任務狀態；節點需 validator 檢查過的證據才能標記完成。 real
State + repository checkpoints 每次節點完成 atomically 寫入 state checkpoint；repo checkpoint 擷取 worktree、做 secret redaction、從不 commit 你的分支。 real
Graceful Pause 核心 durable 持久化 + 無人值守 daemon loop（auspex daemon）+ 四路重驗 resume（repo · quota · session · authorization）—— implemented & tested。 tested

Tier 2 — Vertical slice（已建置＋單元測試，未端到端接線）

能力狀態參照
自動 quota-runway 觸發器 讓 Graceful Pause hands-free 觸發 —— built + unit-tested，尚未在 shipped binary 端到端接線。 slice
Provider turn-interrupt 供應商層的 turn 中斷，用來 hands-free 觸發 pause —— 同上，slice 未接線。 slice

Tier 3 — 已設計・在 M4/M10 roadmap（尚未接線）

設計出處：ADD §13.6 / §18.9。這一層就是「把真實決策接成自動 hook」。

要接的 hook作用Issue

PreCompact checkpoint 壓縮前自動 checkpoint，讓「supervises compaction」字面成真。 #114
